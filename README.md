# MyGamePack Bridge UI

**バージョン: 1.0.2**

TikTok LIVE の配信と Minecraft を連動させるためのアプリです。
視聴者がギフトを送ったり、いいね・シェア・フォロー・訪問をすると、Minecraft の中でコマンドが自動で実行されます。

## どんなことができるの？

TikTok ライブ配信中に視聴者が行ったアクションに応じて、Minecraft の中でコマンドが自動実行されます。

| 視聴者のアクション | Minecraft での動作の例 |
|---|---|
| 🎁 ギフトを送る | ギフトの種類に応じてモブを召喚、アイテムを与えるなど |
| 👍 いいねを押す | 一定数ごとに発動（例：10いいねごとに猫が出現） |
| 🔗 シェアする | シェアのたびに発動（例：スライムが出現） |
| ➕ フォローする | フォローのたびに発動（例：クリーパーが出現） |
| 🚶 ライブに訪問する | 訪問のたびに発動（例：ビレジャーが出現） |
| 💬 コメントする | VOICEVOX でコメントを音声読み上げ |

コマンドの内容はすべて自由にカスタマイズできます！

## 使う前に必要なもの

すべてのものが揃っていないと正しく動きません。事前に確認してください。

### 必須

| 必要なもの | 確認方法・入手先 |
|---|---|
| **Windows 10 / 11（64ビット）** | スタートメニュー → 設定 → システム → バージョン情報で確認 |
| **Node.js** | コマンドプロンプトで `node -v` と入力。バージョンが表示されればOK。表示されない場合は [nodejs.org](https://nodejs.org/) からインストール |
| **TikTok アカウント** | ライブ配信できる状態のもの |

### 任意（あると便利）

| もの | 用途 |
|---|---|
| **VOICEVOX** | コメントやギフトを音声読み上げする |
| **AivisSpeech** | VOICEVOXの代替。どちらか1つでOK |

> **Node.js のインストール方法：**
> 1. [https://nodejs.org/](https://nodejs.org/) を開く
> 2. 「推奨版（LTS）」のダウンロードボタンをクリック
> 3. ダウンロードした `.msi` ファイルを実行して、指示通りに進める
> 4. インストール後、コマンドプロンプトで `node -v` を実行して確認

## インストール方法

1. `MyGamePack Bridge UI Setup 1.0.2.exe` をダブルクリック
2. 「このアプリがデバイスに変更を加えることを許可しますか？」→ **「はい」** をクリック
3. インストール先を確認して **「インストール」** をクリック
4. 完了したら **「完了」** をクリック

インストールが終わると、デスクトップにショートカットが作成されます。

## 初回セットアップ（最初の1回だけ）

アプリを初めて起動すると、自動的に**セットアップ画面**が開きます。
（2回目以降は表示されません）

### ステップ 1：インストール先フォルダを選ぶ

1. **「フォルダを選択」** ボタンをクリック
2. Minecraft サーバーを置きたいフォルダを選んで **「フォルダーの選択」** をクリック

> ⚠️ **注意：** フォルダのパスに**日本語・全角文字・空白**が含まれていると動作しない場合があります。
> 例えば `C:\MinecraftServer` のような英数字のみのパスを推奨します。
> ❌ 悪い例：`C:\ゲーム\サーバー`
> ✅ 良い例：`C:\MinecraftServer`

### ステップ 2：インストールを実行する

1. フォルダが選択されたら **「インストール」** ボタンをクリック
2. サーバーのファイルがコピーされます（少し時間がかかります）

### ステップ 3：初期セットアップを実行する

1. **「セットアップを実行」** ボタンをクリック
2. 黒いウィンドウ（コマンドプロンプト）が開きます
3. 画面の指示に従って進めてください
4. セットアップが終わったら黒いウィンドウを閉じてください

### ステップ 4：完了を確認する

1. **「完了を確認」** ボタンをクリック
2. 「セットアップ完了！」と表示されたら成功です
3. ダッシュボードが使えるようになります 🎉

## 毎回の使い方

配信するたびに以下の手順で起動してください。

```
[ステップ1] アプリを起動する
     ↓
[ステップ2] 「ギフト設定」でTikTokユーザー名を確認する
     ↓
[ステップ3] ダッシュボードで「サーバー起動」をクリック
     ↓
[ステップ4] 黒いウィンドウに「Done!」が表示されるまで待つ（1〜2分）
     ↓
[ステップ5] ダッシュボードで「BRIDGE 起動」をクリック
     ↓
[ステップ6] TikTok でライブ配信を開始する
     ↓
[ステップ7] 視聴者のアクションに反応して Minecraft が動く！
```

> ⚠️ **重要：** サーバーが完全に起動してから（黒いウィンドウに `Done!` が表示されてから）BRIDGE を起動してください。
> 逆の順番だと接続に失敗します。

## 各画面の説明

### ダッシュボード

アプリのメインページです。

| ボタン | 説明 |
|---|---|
| **サーバー起動** | Minecraft Forge サーバーを起動します |
| **BRIDGE 起動** | TikTok と Minecraft を繋ぐブリッジを起動します |

---

### ギフト設定

TikTok のギフトと Minecraft のコマンドを紐付ける設定画面です。

#### 基本設定（TikTok ユーザー名など）

画面の「設定 & DL」タブを開くと設定できます。

| 項目 | 説明 |
|---|---|
| **TikTok ユーザー名** | 配信するアカウントのユーザー名（`@` は不要） |
| **RCON パスワード** | サーバーとの接続パスワード（セットアップ時に自動生成）|

#### ギフトへのコマンド割り当て

1. 上部のギフト一覧から設定したいギフトのカードをクリック
   - 💎マーク = ギフトのコイン数
   - 「済」バッジ = 設定済み
   - 「未」バッジ = 未設定
2. 「実行されるイベント」でコマンドファイルを選択
3. 「繰り返し回数」を設定（高額ギフトほど大きくするのがおすすめ）
4. **「保存」** をクリック

> **繰り返し回数の目安：**
> - 1コイン相当のギフト → 1〜3回
> - 100コイン相当のギフト → 10〜30回
> - 高額ギフト → 50〜100回

ギフト一覧が古い場合は **「🔄 更新」** ボタンで最新データを取得できます。

---

### コマンド設定

コマンドファイル（.txt）の中身をアプリ上で作成・編集できます。

```
# TITLE: ハスク！
# SUBTITLE: ハスクが降り注ぐ
# CATEGORY: 敵対MOB

execute as @p at @p run summon minecraft:husk ^ ^ ^
```

- `# TITLE:` に書いた名前がプルダウンに表示されます
- `#` で始まる行はコメント（実行されません）
- 1行に1コマンドを書きます

作成後は **「TXTをDL」** でダウンロードして、`bridge/commands/minecraft/` フォルダに置いてください。

---

### イベント設定

ギフト以外のアクションにコマンドを割り当てます。

#### 👍 いいねイベント

いいねが「しきい値」の倍数に達するたびに発動します。

| 項目 | 説明 |
|---|---|
| **しきい値** | 何いいねごとに発動するか（例：10 → 10、20、30…いいねで発動） |
| **実行するコマンド** | 発動するコマンドファイルを選択 |
| **有効/無効** | このイベントを使うかどうか |

複数の設定を追加できます（例：10いいねで猫、50いいねでドラゴン）。

#### 🔗 シェア / ➕ フォロー / 🚶 訪問イベント

それぞれのアクションが発生するたびに1回発動します。

| 項目 | 説明 |
|---|---|
| **実行するコマンド** | 発動するコマンドファイルを選択 |
| **有効/無効** | このイベントを使うかどうか |

設定後は **「保存する」** をクリックしてください。
変更はBRIDGEの**次回起動時**に反映されます。

---

### ギフト一覧

TikTok のギフト情報（画像・名前・コイン数）を一覧表示します。
**「🔄 更新」** で最新のギフトデータを取得できます。

---

### ワールド設定

Minecraft サーバーの設定（`server.properties`）をGUIで編集できます。

| 項目 | 説明 |
|---|---|
| **ワールド** | 使用するワールドのフォルダを選択 |
| **ゲームモード** | サバイバル / アドベンチャー / クリエイティブ / スペクテイター |
| **難易度** | ピースフル / イージー / ノーマル / ハード |
| **最大プレイヤー数** | 同時接続できる人数の上限 |

変更後は **「保存」** を押してください。変更はサーバーの**次回起動時**に反映されます。

---

### 読み上げ設定（TTS）

VOICEVOX / AivisSpeech を使って、コメントやギフトを音声読み上げする機能の設定です。

| 項目 | 説明 |
|---|---|
| **エンジン** | VOICEVOX または AivisSpeech を選択 |
| **話者ID** | 読み上げに使うキャラクターのID |
| **読み上げ速度** | 音声の速さ（1.0が標準） |
| **コメント読み上げ** | 視聴者コメントを読み上げるか |
| **ギフト読み上げ** | ギフト通知を読み上げるか |
| **ギフトテンプレート** | 読み上げる文章（例：`{sender}さんから{gift}が来たよ！`） |

> **前提：** VOICEVOX または AivisSpeech を**先に起動してから**、BRIDGEを起動してください。

## コマンドファイルの仕組み

コマンドファイルは `bridge/commands/minecraft/` フォルダ内の `.txt` ファイルです。
ファイルに書かれたコマンドが、Minecraft の RCON 経由で順番に実行されます。

### ファイルの構造

```
# TITLE: 猫ちゃん登場
# SUBTITLE: プレイヤーの近くに猫ちゃん召喚
# CATEGORY: 友好MOB

execute as @p at @p run summon minecraft:cat ^ ^ ^
effect give @e[type=minecraft:cat,sort=nearest,limit=30] minecraft:slow_falling 3 1 true
```

| 種類 | 書き方 | 説明 |
|---|---|---|
| タイトル | `# TITLE: テキスト` | ゲーム画面に表示されるタイトル |
| サブタイトル | `# SUBTITLE: テキスト` | ゲーム画面に表示されるサブタイトル |
| カテゴリ | `# CATEGORY: テキスト` | 管理用のメモ（ゲームには表示されない） |
| コメント行 | `# テキスト` | 実行されない説明文 |
| コマンド行 | `execute as @p ...` など | Minecraft に送るコマンド（1行1コマンド） |

### `{ListenerName}` — 視聴者の名前を表示する

コマンド内に `{ListenerName}` と書くと、ギフトを送った視聴者の名前に自動で置き換わります。

```
# 例：ゾンビに視聴者の名前をつける
summon minecraft:zombie ~ ~ ~ {CustomName:"\"{ListenerName}\"",CustomNameVisible:1b}
```

コマンドファイルの置き場所：

```
bridge/
  commands/
    minecraft/
      cat.txt         ← 猫ちゃん登場
      zombie.txt      ← ゾンビ襲来！
      creeper.txt     ← クリーパー！
      slime.txt       ← スライム！
      villager.txt    ← 村人現る！
      ... （自分で追加できます）
```

## コマンドファイルを自作する

### アプリ上で作る方法（簡単）

1. 「コマンド設定」タブを開く
2. 「+ 新規作成」ボタンをクリック
3. コマンドを入力する
4. **「TXTをDL」** でダウンロード
5. ダウンロードしたファイルを `bridge/commands/minecraft/` フォルダに置く

### メモ帳などで直接作る方法

1. メモ帳を開く
2. 以下のように入力する：

```
# TITLE: 花火大会
# SUBTITLE: 花火が打ち上がる！
# CATEGORY: 演出

summon minecraft:firework_rocket ~ ~5 ~
```

3. 「名前を付けて保存」で、**ファイル名を `hanabi.txt`** のように `.txt` で保存
   - ファイル名は**英数字と `_`（アンダーバー）のみ**にしてください
4. 保存したファイルを `bridge/commands/minecraft/` フォルダに移動

### よく使うコマンドの例

```bash
# プレイヤーの近くにモブを召喚する
execute as @p at @p run summon minecraft:zombie ^ ^ ^1

# プレイヤーにアイテムを与える
give @p minecraft:golden_apple 1

# プレイヤーを回復する
effect give @p minecraft:instant_health 1 10 true

# 雷を落とす
execute as @p at @p run summon minecraft:lightning_bolt

# 空腹状態にする
effect give @p minecraft:hunger 30 5 false

# タイトルメッセージを表示する
title @a title {"text":"ようこそ！","color":"gold"}
```

## よくある質問・トラブルシューティング

### Q. ブリッジを起動してもTikTokに繋がらない

以下を順番に確認してください：

1. **TikTok ユーザー名が正しいか確認する**（`@` は不要・大文字小文字も注意）
2. **現在TikTokでライブ配信中か確認する**（配信していないと接続できません）
3. **インターネットに繋がっているか確認する**

---

### Q. サーバーが起動しない

1. **Node.js がインストールされているか確認する**
   - コマンドプロンプトを開いて `node -v` と入力
   - バージョン番号が表示されればOK（例：`v20.11.0`）
   - 表示されない場合は [nodejs.org](https://nodejs.org/) からインストール
2. **セットアップが完了しているか確認する**
3. **ウイルス対策ソフトがブロックしていないか確認する**
   - Windows Defender や他のセキュリティソフトの除外設定にフォルダを追加してみてください

---

### Q. ギフトを送ってもMinecraftが反応しない

以下を順番に確認してください：

1. **サーバーが起動しているか確認する**（黒いウィンドウに `Done!` が表示されているか）
2. **BRIDGEが起動しているか確認する**
3. **TikTokでライブ配信中か確認する**
4. **該当のギフトがギフト設定に登録されているか確認する**（「済」バッジがついているか）
5. **コマンドファイルが正しいフォルダに置かれているか確認する**

---

### Q. 読み上げが機能しない

1. **VOICEVOX または AivisSpeech を先に起動してから** BRIDGEを起動してください
2. 読み上げ設定で **「コメント読み上げ」や「ギフト読み上げ」が有効**になっているか確認してください
3. 読み上げ設定の **「エンジン」** が実際に使っているソフト（VOICEVOX or AivisSpeech）と一致しているか確認してください

---

### Q. コマンドファイルを変更したのに反映されない

コマンドファイルはBRIDGEの起動時に読み込まれます。
ファイルを変更した後は、BRIDGEを**一度終了して再起動**してください。

---

### Q. 設定を変えたのに反映されない

「保存」ボタンを押したか確認してください。
また、BRIDGEを再起動することで確実に反映されます。

---

### Q. 新しいギフトが一覧に表示されない

「ギフト一覧」画面の **「🔄 更新」** ボタンをクリックすると、TikTokから最新のギフトデータを取得します。

## 開発情報

- **フレームワーク：** Electron + React + Vite + Tailwind CSS
- **TikTok 接続：** `tiktok-live-connector`
- **Minecraft 接続：** RCON（`rcon-client`）
- **音声読み上げ：** VOICEVOX / AivisSpeech（HTTP API）
